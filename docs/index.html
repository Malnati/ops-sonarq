GitHub Composite Action

Malnati/ops-sonarq

Documentation aligned with the current action.yml. This action runs SonarQube scan, collects report files under path/.sonarq, commits the report to a new branch, attempts to open a PR, and uploads an artifact.

Repository: github.com/Malnati/ops-sonarq

Current workflow behavior

  1. Checkout repository.
  2. Resolve scan path from inputs.path (fallback api).
  3. Detect SonarQube host and wait until status is UP.
  4. Create sonar-project.properties from template.
  5. Install SonarScanner CLI and run scan.
  6. Fetch quality gate, metrics, issues, hotspots and analyses JSON files.
  7. Generate REPORT.md.
  8. Commit reports in a new branch and try creating a pull request.
  9. Upload artifact sonarqube-report.

Inputs

Input Required Default Description
path yes api Path to scan
project_key no nome-do-projeto Project key
project_name no Nome do Projeto Project name

Declared outputs

Output Description in action.yml
json Generated JSON file path (array).
report_path Generated JSON file path (array).
status Scan status.
count Number of literals found.

Note: outputs reference steps.generate_output.outputs.*, but there is currently no step with id: generate_output.

Generated files

All files are written to path/.sonarq:

  • quality-gate.json
  • metrics.json
  • issues.json
  • hotspots.json
  • analyses.json
  • REPORT.md

Usage example

- name: Run ops-sonarq
  uses: Malnati/ops-sonarq@v1.0.0
  with:
    path: api
    project_key: meu-projeto
    project_name: Meu Projeto
